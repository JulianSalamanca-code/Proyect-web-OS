Archivos del sistema

CMD sfc /scannow
Descripción

Escanea todos los archivos protegidos del sistema y reemplaza las versiones dañadas o incorrectas por copias correctas almacenadas en caché por Windows.

Sintaxis / Salida
sfc /scannow
Beginning system scan... Verification 100% complete. Windows Resource Protection did not find any integrity violations.
Análisis
SFCSystem File Checker es una herramienta integrada desde Windows 98. El flag /scannow inicia un análisis completo en tiempo real. Requiere privilegios de administrador. Si detecta archivos corruptos que no puede reparar, genera un log en %WinDir%\Logs\CBS\CBS.log para diagnóstico adicional.

Procesador y Sistema

CMD systeminfo
Descripción

Muestra un resumen completo del equipo: nombre del host, versión del SO, fabricante, modelo, procesador, memoria física y parches instalados.

Sintaxis / Salida
systeminfo
Host Name: DESKTOP-AB123CD OS Name: Windows 11 Pro OS Version: 10.0.22631 Processor: Intel Core i7 ~2600 MHz Total RAM: 16,384 MB Available RAM: 8,245 MB
Análisis
WMIConsulta múltiples fuentes del sistema (WMI, registro, APIs de Windows) para consolidar información de hardware y SO en una sola vista. Útil como primer paso en diagnóstico: permite identificar de inmediato la versión exacta del sistema, la cantidad de RAM disponible y los hotfixes aplicados (Hotfix(s) installed).
CMD perfmon /report
Descripción

Genera un informe completo de diagnóstico de rendimiento del sistema, recolectando métricas durante 60 segundos antes de presentar los resultados.

Sintaxis / Salida
perfmon /report
Collecting data for 60 seconds... Analyzing results... Warnings: - High CPU usage detected - 1 device driver is outdated System responsiveness: Good Disk I/O: Normal
Análisis
PerfMonLanza el Monitor de rendimiento de Windows y genera un informe HTML interactivo. Analiza CPU, disco, red, memoria y controladores. El reporte final se abre en el navegador predeterminado y contiene secciones de advertencias, diagnósticos e información del sistema. Muy útil para detectar cuellos de botella sin herramientas de terceros.

Disco duro

CMD chkdsk C: /f /r
Descripción

Verifica la integridad del sistema de archivos y los sectores físicos del disco. /f corrige errores y /r localiza sectores defectuosos recuperando información legible.

Sintaxis / Salida
chkdsk C: /f /r
The type of the file system is NTFS. Stage 1: Examining basic file system... Stage 2: Examining file name linkage... Stage 3: Examining security descriptors... Windows has scanned the file system and found no problems.
Análisis
NTFSOpera a nivel de sistema de archivos y superficie de disco. El flag /f corrige errores lógicos en la tabla de archivos; /r hace una prueba de superficie sector a sector (proceso lento). En unidades en uso, el análisis se programa para el próximo reinicio. Imprescindible ante errores de escritura o sectores dañados.
CMD fsutil dirty query C:
Descripción

Comprueba si el volumen tiene el bit "sucio" activo, lo que indica que Windows programará un chkdsk automático en el próximo arranque.

Sintaxis / Salida
fsutil dirty query C:
Volume - C: is NOT Dirty Volume - C: is Dirty
Análisis
fsutilEl dirty bit es una marca que Windows activa cuando detecta una inconsistencia en el sistema de archivos (apagado abrupto, error de escritura). Si el volumen está "sucio", chkdsk se ejecutará automáticamente al reiniciar. Este comando permite verificar el estado sin necesidad de reiniciar el sistema.

Memoria RAM

CMD mdsched.exe
Descripción

Abre la herramienta de Diagnóstico de Memoria de Windows, que realiza pruebas exhaustivas de los módulos RAM al reiniciar el sistema.

Sintaxis / Salida
mdsched.exe
Windows Memory Diagnostic [•] Restart now and check for problems [ ] Check for problems next startup
Análisis
MdSchedEjecuta pruebas de memoria fuera del entorno de Windows (antes de cargar el SO) para mayor fiabilidad. Ofrece dos niveles: Básico y Extendido (más patrones de prueba). Los resultados se muestran al volver a iniciar sesión. Ideal ante síntomas de pantallas azules (BSOD) o errores aleatorios de aplicaciones.

Programas instalados y actualizaciones

PS Get-WmiObject Win32_Product | Select Name, Version
Descripción

Lista todos los programas instalados en el sistema mediante la clase WMI Win32_Product, mostrando nombre y versión de cada uno.

Sintaxis / Salida
Get-WmiObject Win32_Product ` | Select Name, Version
Name Version ---- ------- Google Chrome 121.0.0 Microsoft Office 365 16.0.0 Git 2.44.0
Análisis
WMIWin32_Product consulta el instalador de Windows (MSI) directamente. Nota: este cmdlet puede ser lento porque valida cada paquete durante la consulta. Para listados rápidos, se prefiere Get-Package o consultar el registro. El operador | (pipeline) pasa los objetos al filtro Select.
PS Get-Package
Descripción

Lista los paquetes y aplicaciones instaladas usando PackageManagement (OneGet), incluyendo fuentes como MSI, Chocolatey o winget.

Sintaxis / Salida
Get-Package
Name Version Source ---- ------- ------ 7zip 19.00 Programs Git 2.44.0 Programs VSCode 1.87.0 Programs
Análisis
PackageMgmtA diferencia de Win32_Product, Get-Package es significativamente más rápido al no lanzar el proceso de validación MSI. Permite filtrar por proveedor con -ProviderName o buscar un paquete específico con -Name "nombre". Útil para auditorías de software.
PS Get-WindowsUpdateLog
Descripción

Convierte los registros ETL de Windows Update en un archivo de texto legible (.log) y lo guarda en el escritorio del usuario actual.

Sintaxis / Salida
Get-WindowsUpdateLog
Windows Update log successfully written to: C:\Users\Admin\Desktop\ WindowsUpdate.log
Análisis
ETWWindows almacena los registros de actualización en formato binario ETL (Event Tracing for Windows). Este cmdlet los convierte a texto plano usando símbolos de depuración. El log resultante permite rastrear descargas fallidas, errores de instalación y el historial completo de parches aplicados al sistema.

Procesador y Sistema

PS Get-WmiObject Win32_Processor | Select Name, NumberOfCores…
Descripción

Obtiene información detallada del procesador: nombre, número de núcleos físicos, hilos lógicos y velocidad máxima de reloj.

Sintaxis / Salida
Get-WmiObject Win32_Processor ` | Select Name, NumberOfCores, NumberOfLogicalProcessors
Name : Intel Core i7-10750H NumberOfCores : 6 LogicalProcessors : 12 MaxClockSpeed : 2600
Análisis
CIM/WMILa clase Win32_Processor expone más de 40 propiedades del CPU. La diferencia entre NumberOfCores (físicos) y NumberOfLogicalProcessors (hilos) indica si el procesador usa Hyper-Threading. Aquí 6 núcleos → 12 hilos lógicos confirma HT activo. Útil para ajustar procesos de paralelización.
PS Get-ComputerInfo
Descripción

Devuelve un objeto con más de 200 propiedades del sistema: versión de Windows, nombre del equipo, memoria total, número de procesadores y mucho más.

Sintaxis / Salida
Get-ComputerInfo
WindowsProductName : Windows 11 Pro WindowsVersion : 22H2 CsName : DESKTOP-AB123CD CsTotalPhysicalMemory: 17179869184 CsProcessorCount : 1
Análisis
CIMIntroducido en PowerShell 5.1, reemplaza múltiples consultas WMI por una sola llamada consolidada. CsTotalPhysicalMemory devuelve bytes (17 179 869 184 B = 16 GB). Para filtrar propiedades específicas se puede combinar con Select-Object -Property "Cs*". Muy eficiente para inventario automatizado de equipos.

Memoria RAM

PS Get-Counter '\Memory\Available MBytes'
Descripción

Consulta en tiempo real el contador de rendimiento de memoria disponible, devolviendo el valor actual en megabytes junto al timestamp de la medición.

Sintaxis / Salida
Get-Counter '\Memory\Available MBytes'
Timestamp CounterSamples --------- -------------- 19/04/2026 10:15 \\DESKTOP\memory\ available mbytes: 8245
Análisis
PerfCounterAccede directamente a los contadores de rendimiento de Windows (misma fuente que el Monitor de rendimiento). Para monitoreo continuo se puede usar el parámetro -Continuous o -SampleInterval 5 -MaxSamples 12 para tomar 12 muestras cada 5 segundos. Ideal para scripts de alerta de memoria en servidores.
